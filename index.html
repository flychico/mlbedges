The methodology, in plain English

Every morning, before any picks are published, the model pulls the day's real MLB schedule, current standings, and probable starters, then works through the same process for every game on the slate. First, each team gets a strength number blending its Pythagorean win expectation — a well-established estimate of "true" win percentage derived from runs scored and allowed, which tends to be more stable than raw record — with its last-10-game form, weighted so a hot or cold recent stretch nudges the number without overwhelming the full-season signal. Those two team strengths are combined head-to-head using a log5 calculation, the standard method for estimating one team's win probability against a specific opponent, and the home team gets a modest home-field adjustment reflecting MLB's roughly 54% historical home win rate.

From there, the starting pitching matchup shifts the number. Each starter's season ERA is compared to league average, with a floor: any starter under 20 innings pitched for the season is treated as league-average rather than trusted on a small, noisy sample, and extreme ERA values are clamped into a realistic range so one outlier start can't distort the projection. The size of the shift is proportional to the ERA gap between the two starters — see the full breakdown in the pitching metrics guide.

Total runs and run line use a separate, lighter model: each team's scoring rate and its opponent's run prevention are combined, adjusted for the home ballpark's park factor (see the park factors guide) and, for open-air stadiums, the game-time weather forecast, then compared to the market line through a standard normal-distribution approximation. A run-line pick is only published if it's directionally consistent with the same game's moneyline projection — see run line vs. moneyline for why that consistency check matters.

Every one of these three numbers — moneyline, total, run line — is then compared to the sportsbook market's no-vig (fair) probability, not the raw price. See the no-vig odds calculator guide for exactly how that works. Only when the model's probability clears the market's fair number by a meaningful margin does a game get labeled a play; everything else is marked an explicit pass. That's what "no edge, no play" means in practice — read more on what edge actually means and how to find value in MLB moneylines.

Frequently asked questions

How does LyDia's MLB model work?

LyDia blends each team's Pythagorean win expectation (from run differential) with last-10 form, combines the two teams head-to-head with a log5 calculation and a home-field adjustment, then shifts the number based on the starting pitching matchup using each starter's season ERA (with a floor for low-innings starters). Totals and run lines use a separate normal-approximation model on the same team and pitcher inputs, adjusted for ballpark and weather.

Is LyDia's model always right?

No, and it isn't trying to be. The model publishes a probability and compares it to the market's fair price — when the two are close, or the model has no real edge, it explicitly passes rather than forcing a play. Every graded pick, win or lose, stays visible on the public Results page.
